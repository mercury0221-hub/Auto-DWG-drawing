## 1. 이 프로그램이 하는 일

```
사용자 → 한국어로 명령 → AI(LLM) → 좌표 계산 → AutoCAD에 직접 작도
```

예를 들어 웹 화면에 **"건물 내부에 3000×2000mm 사각형 방을 그려줘"** 라고 입력하면:

1. AI가 도면의 건물 위치를 파악하고
2. 사각형의 좌표를 계산한 뒤
3. 열려 있는 AutoCAD 도면에 직접 그려줍니다

**지원하는 도형:** 직선(line), 폴리라인(polyline — 사각형 등 연결된 선)

## 2. 사용하기 전에 필요한 것

| 항목 | 확인 방법 |
|---|---|
| Windows 운영체제 | AutoCAD COM 자동화는 Windows 전용 |
| AutoCAD (유료 버전) | 설치 및 실행 가능해야 함 |
| Python 3.x | 시작 메뉴 → `python --version` 입력 |
| OpenRouter 계정 및 API 키 | [openrouter.ai](https://openrouter.ai) 에서 무료 가입 |
| 인터넷 연결 | AI 모델 호출 필요 |

### OpenRouter API 키 발급 방법

1. [https://openrouter.ai](https://openrouter.ai) 접속
2. 우측 상단 **Sign in** → Google/GitHub 계정으로 가입
3. 로그인 후 상단 메뉴 **Keys** 클릭
4. **Create Key** 버튼 클릭 → 키 이름 입력 → **Create**
5. 생성된 키(`sk-or-v1-...` 형태)를 복사해 안전한 곳에 보관

> 무료 계정은 하루 200회, 분당 20회 호출 제한이 있습니다.

## 4. 최초 1회 설정 — .env 파일 만들기

API 키는 프로젝트 폴더의 **`.env` 파일** 하나에 저장해서 관리합니다.  
별도의 암호화 절차나 마스터 키 없이, 파일만 있으면 바로 사용할 수 있습니다.

### 4-1. .env 파일 생성

프로젝트 폴더(`C:\Users\user\OneDrive\바탕 화면\회사\프로젝트\Vibe coding`)에  
메모장(또는 VS Code)으로 **`.env`** 파일을 만들고 아래 내용을 입력합니다.

```
OPENROUTER_API_KEY=sk-or-v1-여기에_본인의_API키_붙여넣기
```

**예시:**
```
OPENROUTER_API_KEY=sk-or-v1-abcd1234efgh5678ijkl9012mnop3456qrst7890uvwx
```

파일을 저장하면 설정이 완료됩니다.  
서버를 시작할 때 이 파일을 자동으로 읽어 API 키를 로드합니다.

### 4-2. .env 파일 위치 확인

```
C:\Users\user\OneDrive\바탕 화면\회사\프로젝트\Vibe coding\
├── .env                ← 이 파일에 API 키 저장
├── server.py
├── proto.py
├── 서버시작.bat
└── ...
```

> `.env` 파일은 외부에 공유하거나 이메일로 전송하지 마세요.  
> API 키가 노출되면 [openrouter.ai](https://openrouter.ai) → Keys 에서 키를 삭제하고 새로 발급받으세요.

## 5. 서버 시작하기

### 방법 A — 배치 파일로 시작 (권장)

프로젝트 폴더에 있는 **`서버시작.bat`** 파일을 더블클릭합니다.

1. `.env` 파일에서 API 키를 자동으로 읽습니다
2. 서버가 시작되면 **브라우저가 자동으로** `http://localhost:5000` 을 엽니다

터미널 창에 아래 메시지가 나오면 서버가 정상 동작 중입니다.

```
================================================
  DWG 자동 작도 서버
  브라우저: http://localhost:5000
================================================
 * Running on http://127.0.0.1:5000
```

### 방법 B — PowerShell로 시작

```powershell
cd "C:\Users\user\OneDrive\바탕 화면\회사\프로젝트\Vibe coding"
python server.py
```

`.env` 파일이 같은 폴더에 있으면 API 키를 자동으로 읽습니다.

### 방법 C — 브라우저 바로가기

서버가 이미 실행 중일 때는 **`DWG작도UI.url`** 을 더블클릭하면  
브라우저에서 `http://localhost:5000` 이 바로 열립니다.

### 서버 종료

서버 터미널 창에서 **Ctrl + C** 를 누릅니다.

## 7. AutoCAD 연결하기

### 연결 전 준비

1. **AutoCAD를 먼저 실행**합니다
2. 작도할 **DWG 파일을 엽니다** (예: `A-0304_test.dwg`)
3. AutoCAD가 **명령 대기 상태**인지 확인합니다
   - 화면 하단에 `Command:` 가 표시되어야 함
   - 팝업창이나 경고창이 없어야 함
   - 명령 실행 중이면 **Esc 키를 2~3번** 눌러 취소

### 연결 확인

웹 화면 상단 헤더에서 연결 상태를 확인합니다.

| 표시 | 의미 |
|---|---|
| 초록 점 + 파일명 | 정상 연결됨 — 작도 가능 |
| 노란 점 (깜빡임) | 연결 확인 중 |
| 빨간 점 + 오류 메시지 | 연결 실패 — 아래 해결 방법 참고 |

**연결 실패 시:** 헤더의 **↺ 새로고침** 버튼을 클릭합니다.

> AutoCAD가 팝업창을 표시 중이면 연결이 거부됩니다.  
> 모든 팝업을 닫고 `Command:` 상태로 만든 뒤 새로고침하세요.

## 8. 그림 그리기 — 작도 명령 입력

### 기본 사용 방법

1. **AutoCAD와 DWG 파일을 열어둡니다** (7번 참고)
2. **서버를 시작합니다** (5번 참고)
3. 브라우저 웹 화면을 엽니다
4. 왼쪽 명령 입력창에 **한국어로 원하는 그림을 설명**합니다
5. **▶ 작도 실행** 버튼을 클릭합니다 (또는 **Ctrl + Enter**)
6. 오른쪽 로그에서 처리 과정을 확인합니다
7. AutoCAD 화면에서 결과를 확인합니다

### 작도 실행 흐름

```
명령 입력
  ↓
[모델] 사용 모델 표시
  ↓
[연결] AutoCAD 연결
  ↓
[도면 분석] 건물 위치 파악 (A-WALL 레이어 경계 추출)
  ↓
[LLM] AI가 건물 내부 좌표 계산
  ↓
[검증] 좌표 형식 확인
  ↓
[작도] AutoCAD에 직접 그리기
  ↓
✓ 완료
```

## 9. AI 모델 선택하기

웹 화면 왼쪽에 모델 선택 라디오 버튼 3개가 있습니다.

### ● 무료 자동선택 (기본값, 권장)

OpenRouter에서 무료 모델을 자동으로 찾아 사용합니다.  
기본 모델: `nvidia/nemotron-3-super-120b-a12b:free`  
**특별한 이유가 없으면 이 옵션을 사용하세요.**

### ○ openrouter/auto

OpenRouter가 가장 적합한 모델을 자동으로 선택합니다.  
유료 모델이 선택될 수 있어 소액의 요금이 발생할 수 있습니다.

### ○ 직접 입력

```
[모델 ID 입력창________________]  [🔍 검색]
```

사용할 AI 모델을 직접 지정합니다.  
무료 모델 한도 초과 시 다른 모델로 전환할 때 사용합니다.

#### 🔍 검색으로 무료 모델 찾기

1. **직접 입력** 라디오를 클릭합니다
2. **🔍 검색** 버튼을 클릭합니다
3. OpenRouter에서 현재 사용 가능한 **무료 모델 전체 목록**이 로드됩니다
4. 검색창에 모델명을 입력해 필터링합니다  
   예: `llama`, `gemma`, `qwen`, `mistral`, `deepseek`
5. 원하는 모델을 클릭하면 자동으로 입력됩니다
6. **▶ 작도 실행** 버튼을 클릭해 해당 모델로 작도합니다

| 모델 목록의 뱃지 | 의미 |
|---|---|
| `vision` | 이미지 인식도 지원하는 모델 |
| `128k`, `200k` | 한 번에 처리 가능한 텍스트 양 (클수록 좋음) |

## 11. API 키 변경 방법

OpenRouter API 키가 변경되거나 새로 발급받은 경우, **.env 파일만 수정**하면 됩니다.

### 변경 절차

**1단계:** 프로젝트 폴더의 `.env` 파일을 메모장으로 엽니다.

```
C:\Users\user\OneDrive\바탕 화면\회사\프로젝트\Vibe coding\.env
```

**2단계:** `OPENROUTER_API_KEY` 값을 새 키로 교체합니다.

```
OPENROUTER_API_KEY=sk-or-v1-새로운키값을여기에붙여넣기
```

**3단계:** 파일을 저장합니다.

**4단계:** 서버를 재시작합니다.
- 실행 중인 서버 창에서 **Ctrl + C** 로 종료
- **`서버시작.bat`** 을 더블클릭해서 다시 시작

변경된 키가 즉시 적용됩니다. 다른 설정은 변경하지 않아도 됩니다.

### 키가 여러 개인 경우

프로젝트별 또는 용도별로 `.env` 파일을 별도로 관리할 수 있습니다.

```
Vibe coding\
  ├── .env              ← 현재 사용 중인 키
  ├── .env.backup       ← 이전 키 백업 (선택사항)
```

사용할 키 파일을 `.env` 로 이름을 바꾸면 해당 키로 전환됩니다.

## 13. 문제 해결 (오류가 날 때)

### 브라우저에 "서버 응답 없음"

**원인:** 웹 화면이 서버가 아닌 파일로 직접 열렸거나, 서버가 꺼진 경우입니다.

**해결:**
1. 브라우저 주소창에 `http://localhost:5000` 이 표시되어 있는지 확인
2. 그렇지 않다면 `서버시작.bat` 을 더블클릭해서 서버를 시작
3. 서버가 이미 실행 중이면 `DWG작도UI.url` 을 더블클릭

---

### AutoCAD 연결 안 됨 (빨간 점)

**원인:** AutoCAD가 명령 대기 상태가 아닌 경우입니다.

**해결 순서:**
1. AutoCAD 창을 클릭해서 활성화
2. **Esc 키를 2~3번** 눌러 현재 명령 취소
3. 화면 하단에 `Command:` 가 보이는지 확인
4. 팝업창이 있으면 모두 닫기
5. 웹 화면에서 **↺ 새로고침** 버튼 클릭

---

### "피호출자가 호출을 거부했습니다" 오류

**원인:** AutoCAD가 바쁜 상태(명령 실행 중, 저장 중 등)입니다.

**해결:**
- AutoCAD에서 모든 작업을 마친 뒤 `Command:` 상태에서 다시 시도하세요
- 시스템이 자동으로 최대 5회 재시도합니다. 잠시 기다리세요

---

### "API 키를 찾을 수 없습니다" 오류

**원인:** `.env` 파일이 없거나 `OPENROUTER_API_KEY` 항목이 빠져 있습니다.

**해결:**
1. 프로젝트 폴더에 `.env` 파일이 있는지 확인
2. `.env` 파일을 메모장으로 열어 아래 내용이 있는지 확인
   ```
   OPENROUTER_API_KEY=sk-or-v1-...
   ```
3. 없으면 추가 후 저장 → 서버 재시작

---

### AI가 건물 밖에 그림

**원인:** AI가 건물 위치를 파악하지 못한 경우입니다.

**해결:**
- 명령에 **"건물 내부에"** 또는 **"건물 안에"** 를 포함하세요
- AutoCAD에서 `A-WALL` 레이어가 켜져 있는지 확인하세요
- 한 번 더 실행하면 도면을 재분석해서 올바른 위치에 그립니다

---

### 무료 모델 API 한도 초과 (429 오류)

**원인:** 하루 또는 분당 호출 횟수 제한을 초과했습니다.

**해결 방법:**
1. 분당 한도 초과 → 1분 후 재시도
2. 하루 한도 초과 → 다음 날 재시도, 또는 다른 무료 모델로 전환
3. 다른 무료 모델 전환: **직접 입력** 선택 → **🔍 검색** 클릭 → 다른 모델 선택
4. `openrouter/auto` 옵션 사용 (유료 모델 포함, 소액 요금 가능)

---

### JSON 파싱 오류

**원인:** AI 응답이 잘못된 형식으로 왔습니다.

**해결:**
- 명령을 다시 입력해서 재시도하세요
- 명령을 더 구체적으로 작성해보세요 (크기와 위치를 명확히)

## 14. 용어 설명

| 용어 | 설명 |
|---|---|
| **DWG** | AutoCAD에서 사용하는 도면 파일 형식 |
| **LLM** | Large Language Model. 자연어를 이해하는 AI (ChatGPT와 같은 종류) |
| **OpenRouter** | 여러 AI 회사의 모델을 한 곳에서 사용할 수 있게 해주는 서비스 |
| **API 키** | OpenRouter 서비스를 사용하기 위한 인증 비밀번호 (`sk-or-v1-...`) |
| **.env 파일** | API 키 등 설정값을 저장하는 텍스트 파일. 서버 시작 시 자동으로 읽힘 |
| **COM 자동화** | 프로그램이 AutoCAD를 코드로 제어하는 Windows 기술 |
| **폴리라인** | 여러 점을 연결한 선. 사각형 등 다각형 그릴 때 사용 |
| **SSE** | Server-Sent Events. 서버에서 실시간으로 정보를 전송하는 방식 |
| **localhost:5000** | 내 컴퓨터에서 실행 중인 서버 주소 (인터넷이 아닌 내부 주소) |
| **A-WALL 레이어** | AutoCAD 도면에서 벽(Wall)이 그려진 레이어 이름 |
| **건물내부영역** | AI에게 전달되는 건물 경계 좌표 범위 |
| **작도** | 도면에 선이나 도형을 그리는 것 |
| **좌표 (x, y)** | 도면 위의 위치. x는 가로, y는 세로 방향 위치값 (단위: mm) |

## 빠른 시작 요약

### 처음 사용 시 (1회만)

```
1단계: pip install pyautocad openai cryptography requests flask

2단계: 프로젝트 폴더에 .env 파일 생성
       내용: OPENROUTER_API_KEY=sk-or-v1-본인의키

3단계: 서버시작.bat 더블클릭 → 브라우저 자동 오픈
```

### 매번 사용 시

```
1단계: AutoCAD 실행 → DWG 파일 열기 → Command: 대기 상태 확인

2단계: 서버시작.bat 더블클릭 → 브라우저 자동 오픈

3단계: 웹 화면 상단 초록 점 + 파일명 확인 (AutoCAD 연결됨)

4단계: 명령 입력창에 한국어로 입력 → ▶ 작도 실행 클릭

5단계: AutoCAD 화면에서 결과 확인
```

### API 키 변경 시

```
1단계: .env 파일을 메모장으로 열기

2단계: OPENROUTER_API_KEY= 값을 새 키로 교체 후 저장

3단계: 서버 종료(Ctrl+C) → 서버시작.bat 재실행
```
